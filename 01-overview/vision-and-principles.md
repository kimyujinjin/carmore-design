# Vision & Principles

## 비전

> **"여행의 모든 순간을 명확하게"**

Carmore Design System은 전 세계 사용자가 렌터카와 숙박을 탐색하고 예약하는 모든 여정에서 **명확하고 신뢰할 수 있는 경험**을 제공합니다. 복잡한 예약 과정을 단순하게, 낯선 환경에서도 익숙하게 느낄 수 있도록 일관된 디자인 언어를 구축합니다.

---

## 핵심 원칙

### 1. Clarity (명확성)

여행 예약은 날짜, 가격, 위치, 옵션 등 정보가 많은 도메인입니다. 모든 UI 요소는 사용자가 **한눈에 핵심 정보를 파악**할 수 있도록 설계합니다.

**실천 기준:**
- 가격, 날짜 등 핵심 정보는 시각적 위계(hierarchy)에서 가장 먼저 인지되어야 합니다
- 한 화면에 하나의 주요 액션(Primary CTA)만 배치합니다
- 상태(예약 확정, 대기 중, 취소됨)는 색상과 아이콘을 함께 사용하여 전달합니다

### 2. Consistency (일관성)

Web, iOS, Android 세 플랫폼에서 **동일한 토큰과 컴포넌트 언어**를 공유하여, 사용자가 어떤 디바이스에서든 같은 브랜드 경험을 느끼도록 합니다.

**실천 기준:**
- 모든 시각 속성은 디자인 토큰으로 정의하고, 하드코딩된 값을 사용하지 않습니다
- 동일한 역할의 컴포넌트는 플랫폼과 무관하게 같은 이름과 동작을 가집니다
- 플랫폼 네이티브 관습(iOS 제스처, Android 백 버튼 등)은 존중하되, 브랜드 정체성은 유지합니다

### 3. Accessibility (접근성)

글로벌 서비스로서 다양한 사용자 환경을 지원합니다. 색각 이상, 저시력, 모터 장애, 스크린 리더 사용자 모두가 불편 없이 예약을 완료할 수 있어야 합니다.

**실천 기준:**
- 모든 텍스트-배경 조합은 WCAG 2.1 AA 이상의 명도 대비(4.5:1)를 충족합니다
- 색상만으로 정보를 전달하지 않고, 텍스트 라벨이나 아이콘을 병행합니다
- 모든 인터랙티브 요소는 최소 터치 타겟 44×44px (iOS) / 48×48dp (Android)를 준수합니다
- 키보드 및 스크린 리더 탐색을 지원합니다

### 4. Efficiency (효율성)

디자인 시스템은 제품 팀의 **속도를 높이는 도구**입니다. 반복적인 의사결정을 줄이고, 기획자-디자이너-개발자 간의 해석 차이를 최소화합니다.

**실천 기준:**
- 컴포넌트는 가장 흔한 사용 사례를 기본값(default)으로 제공합니다
- 네이밍 규칙은 역할과 의도를 명확히 반영하여 별도 설명 없이 이해 가능해야 합니다
- 새로운 패턴이 필요할 때, 기존 컴포넌트를 조합(composition)하여 해결할 수 있는지 먼저 검토합니다

### 5. Scalability (확장성)

서비스 성장에 따라 새로운 상품 유형, 플랫폼, 테마가 추가될 수 있습니다. 시스템은 이를 수용할 수 있도록 **계층적이고 모듈화된 구조**를 갖습니다.

**실천 기준:**
- 3단계 토큰 구조(Raw → Scale → Semantic)로 값 변경이 전체 시스템에 안전하게 전파됩니다
- 컴포넌트는 variant와 slot 기반으로 확장하며, 기존 API를 깨뜨리지 않습니다
- 다크 모드, 고대비 모드 등 추가 테마는 Semantic 토큰 레이어에서 대응합니다

## 디자인 시스템이 해결하는 문제

| 문제 | 해결 방식 |
|------|-----------|
| "이 버튼 색상이 정확히 뭐예요?" | `$color.bg.brand` 토큰으로 소통 |
| "iOS와 Android가 왜 다르게 보여요?" | 동일 토큰 기반 플랫폼별 매핑 문서 |
| "이전에 쓰던 카드랑 새 카드가 어떻게 달라요?" | 컴포넌트별 variant 문서와 Do/Don't |
| "새 화면을 만들 때 간격을 어떻게 잡아야 해요?" | Spacing 시맨틱 토큰 + 패턴 가이드 |
| "디자이너가 전달한 스펙을 어떻게 읽어요?" | Handoff Guide + 토큰 참조 규칙 |
