# Card

> 관련 콘텐츠를 그룹화하여 표시하는 컨테이너 컴포넌트

---

## 1. 설명

Card는 관련된 정보를 시각적으로 **하나의 단위로 묶어** 표시하는 컴포넌트입니다. Carmore에서는 검색 결과(listing), 예약 요약(booking), 일반 콘텐츠(base) 등 핵심 정보 표시에 사용됩니다.

Card의 variant는 **콘텐츠의 맥락**에 따라 선택합니다:
- **Base**: 범용 콘텐츠 컨테이너
- **Booking**: 예약 요약 카드 (썸네일, 제목, 날짜, 가격, 상태 배지)
- **Listing**: 검색 결과 카드 (이미지, 제목, 평점, 가격, 주요 특징)

---

## 2. 해부도 (Anatomy)

### variant="base"

```
┌──────────────────────────────────────────┐
│  ①[Content Slot]                         │  ② Container
│                                          │
└──────────────────────────────────────────┘
```

| # | Part | 필수 여부 | 설명 |
|---|------|-----------|------|
| ① | Content Slot | 필수 | 자유롭게 배치 가능한 콘텐츠 영역 |
| ② | Container | 필수 | 배경, 테두리, 그림자를 포함하는 외곽 |

### variant="booking"

```
┌──────────────────────────────────────────┐
│  ┌────┐  ②Title               ⑤[Badge]  │
│  │ ①  │  ③Date Range                    │  ⑥ Container
│  │Thumb│  ④Price                         │
│  └────┘                                  │
└──────────────────────────────────────────┘
```

| # | Part | 필수 여부 | 설명 |
|---|------|-----------|------|
| ① | Thumbnail | 필수 | 예약 대상의 정사각형 썸네일 (1:1) |
| ② | Title | 필수 | 렌터카 모델명 또는 숙소명 |
| ③ | Date Range | 필수 | 예약 기간 (예: "3월 15일 – 3월 18일") |
| ④ | Price | 필수 | 총 결제 금액 |
| ⑤ | Status Badge | 필수 | 예약 상태 ("확정", "대기", "취소") |
| ⑥ | Container | 필수 | 배경, 테두리를 포함하는 외곽 |

### variant="listing"

```
Vertical:
┌──────────────────────────────────────────┐
│  ┌────────────────────────────────────┐  │
│  │           ① Image (16:9)           │  │
│  └────────────────────────────────────┘  │
│  ②Title                                  │
│  ③Location                               │  ⑦ Container
│  ④Rating ★★★★☆ (128)                    │
│  ⑤₩85,000 /박                            │
│  ⑥[WiFi] [주차] [조식]                    │
└──────────────────────────────────────────┘

Horizontal:
┌──────────────────────────────────────────┐
│  ┌──────┐  ②Title                       │
│  │  ①   │  ③Location                    │
│  │Image │  ④Rating ★★★★☆ (128)          │  ⑦ Container
│  │      │  ⑤₩85,000 /박                  │
│  └──────┘  ⑥[WiFi] [주차]                │
└──────────────────────────────────────────┘
```

| # | Part | 필수 여부 | 설명 |
|---|------|-----------|------|
| ① | Image | 필수 | 대표 이미지 (vertical: 16:9, horizontal: 4:3) |
| ② | Title | 필수 | 렌터카 모델명 또는 숙소명 |
| ③ | Location | 선택 | 위치 정보 (예: "제주시 연동") |
| ④ | Rating + Reviews | 선택 | 별점 + 리뷰 수 |
| ⑤ | Price | 필수 | 1박/1일 기준 가격 |
| ⑥ | Feature Tags | 선택 | 주요 특징 태그 (최대 3개) |
| ⑦ | Container | 필수 | 배경, 테두리를 포함하는 외곽 |

## 3. 옵션 (Options)

| Property | Type | Values | Default | 설명 |
|----------|------|--------|---------|------|
| `variant` | enum | `"base"` \| `"booking"` \| `"listing"` | `"base"` | 카드의 시각적 변형 |
| `interactive` | boolean | `true` \| `false` | `true` | 탭/클릭 시 상세 페이지 이동 여부 |
| `selected` | boolean | `true` \| `false` | `false` | 선택 상태 (비교 기능 등에서 사용) |
| `orientation` | enum | `"vertical"` \| `"horizontal"` | `"vertical"` | listing variant의 레이아웃 방향 |
| `status` | enum | `"confirmed"` \| `"pending"` \| `"cancelled"` | — | booking variant의 예약 상태 |

---

## 4. 인터랙션 (Interactions)

| 입력 | 동작 |
|------|------|
| **Tap / Click** | `interactive=true`일 때 상세 페이지로 이동 |
| **Hover** (Web) | 카드 elevation 상승, 커서 pointer로 변경 |
| **Keyboard Focus** | 포커스 링 표시 |
| **Enter / Space** (키보드) | Tap과 동일하게 상세 페이지 이동 |
| **Long Press** (모바일) | 빠른 미리보기 또는 컨텍스트 메뉴 (플랫폼별 상이) |
| **Selected** | 선택 상태 스타일 (stroke 강조) 적용 |

## 5. 가이드라인 (Do & Don't)

### DO

- ✅ **카드 내 메타데이터는 최대 3개까지만 표시합니다**
  ```
  ✅ Rating, Price, Location
  ❌ Rating, Price, Location, Distance, Category, Host
  ```

- ✅ **이미지 비율을 일관되게 유지합니다**
  ```
  listing: 16:9 (vertical), 4:3 (horizontal)
  booking: 1:1 (정사각형 썸네일)
  ```

- ✅ **리스트 내 모든 카드는 동일한 variant와 orientation을 사용합니다**

- ✅ **가격은 항상 단위를 포함합니다** — "/박", "/일" 등

- ✅ **상태 배지(booking)는 항상 우측 상단에 배치합니다**

### DON'T

- ❌ **카드 안에 카드를 중첩(nesting)하지 않습니다**
  ```
  ❌ Card > Card > Content
  ✅ Card > Content
  ```

- ❌ **한 카드에 여러 개의 주요 액션을 넣지 않습니다** — 카드 자체가 하나의 탭 타겟
  ```
  ❌ Card 내부에 [예약] [비교] [찜] 버튼 3개
  ✅ Card 탭 → 상세 페이지에서 액션 수행
  ```

- ❌ **Feature Tag를 4개 이상 표시하지 않습니다** — 핵심 3개만 노출

- ❌ **이미지 없이 listing variant를 사용하지 않습니다** — 이미지가 없으면 base variant 사용

## 6. 플랫폼 차이 (Platform Differences)

| 항목 | Web | iOS | Android |
|------|-----|-----|---------|
| Hover 상태 | elevation 상승 | 미지원 (터치) | 미지원 (터치) |
| listing 기본 방향 | 반응형 (아래 참조) | vertical | vertical |
| 이미지 로딩 | lazy load | 시스템 캐시 | Glide/Coil 캐시 |
| 카드 간 간격 | 16px | 16pt | 16dp |
| 최소 터치 타겟 | 44px | 44pt | 48dp |

### 반응형 레이아웃 (Web)

| Breakpoint | listing orientation | 카드 그리드 |
|------------|-------------------|------------|
| Mobile (< 768px) | vertical | 1열 |
| Tablet (768–1199px) | horizontal | 2열 |
| Desktop (≥ 1200px) | horizontal | 3–4열 |

> Booking variant는 모든 breakpoint에서 horizontal 레이아웃을 유지합니다.

## 7. 접근성 (Accessibility)

| 속성 | 값 |
|------|-----|
| **Role** | `interactive=true`: `link` (상세 이동), `interactive=false`: `article` |
| **aria-label** | 카드의 핵심 정보 요약 (예: "소나타 2024, 3월 15일–18일, 150,000원, 확정") |
| **aria-selected** | selected 상태에서 `true` |
| **alt (이미지)** | 대상을 설명하는 대체 텍스트 (예: "현대 소나타 2024 외관", "해변 앞 리조트 전경") |
| **키보드** | `Tab`으로 포커스, `Enter`로 상세 이동 |
| **스크린 리더** | "[제목], [가격], [상태], 링크" 순서로 읽힘 |
| **색상 외 표시** | 상태 배지는 색상 + 텍스트 라벨로 이중 표시 ("확정", "대기", "취소") |
