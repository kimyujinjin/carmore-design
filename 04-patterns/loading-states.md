# Loading States (로딩 상태 패턴)

> 데이터를 불러오는 동안의 시각적 피드백 패턴

---

## 개요

Loading State는 사용자에게 **시스템이 동작 중**임을 전달하고, 체감 대기 시간을 줄이는 역할을 합니다. 로딩 유형에 따라 적절한 패턴을 선택합니다.

---

## 로딩 유형

| 유형 | 사용 시점 | 패턴 |
|------|-----------|------|
| **Skeleton** | 페이지/섹션 최초 로딩 | 콘텐츠 자리 표시자 |
| **Spinner** | 버튼 클릭 후 처리 중 | 인라인 회전 아이콘 |
| **Progress Bar** | 파일 업로드 등 진행률 표시 | 가로 프로그레스 바 |
| **Pull to Refresh** | 리스트 갱신 (모바일) | 당겨서 새로고침 |
| **Infinite Scroll** | 리스트 추가 로딩 | 하단 스피너 |

---

## 1. Skeleton (스켈레톤)

### 원칙

- 실제 콘텐츠와 **같은 레이아웃**의 회색 자리표시자를 표시합니다
- 콘텐츠가 로드되면 스켈레톤이 실제 콘텐츠로 교체됩니다 (fade 전환)
- 펄스(pulse) 애니메이션으로 로딩 중임을 표시합니다

### 스타일

| 요소 | 토큰 |
|------|------|
| 배경색 | `$color.bg.neutral-weak` (`#F7F8FA`) |
| 펄스 밝은색 | `$color.bg.neutral-hover` (`#F0F1F3`) |
| 애니메이션 | `$motion.duration.slower` (500ms), `$motion.easing.standard`, 무한 반복 |
| 모서리 곡률 | 텍스트 → `$radius.xs` (4px), 이미지 → 원본과 동일 |

### Card (Listing) Skeleton 예시

```
┌─────────────────────────────────┐
│  ┌─────────────────────────┐   │
│  │  ░░░░░░░░░░░░░░░░░░░░░  │   │  ← 이미지 자리 (16:9, $radius.sm)
│  │  ░░░░░░░░░░░░░░░░░░░░░  │   │
│  └─────────────────────────┘   │
│  ░░░░░░░░░░░░                  │  ← 제목 (너비 60%)
│  ░░░░░░░░                      │  ← 위치 (너비 40%)
│  ░░░░░                         │  ← 평점 (너비 25%)
│  ░░░░░░░░░                     │  ← 가격 (너비 35%)
└─────────────────────────────────┘
```

### Card (Booking) Skeleton 예시

```
┌─────────────────────────────────┐
│  ┌──────┐  ░░░░░░░░░░░         │  ← 썸네일 80×80 + 제목
│  │░░░░░░│  ░░░░░░░              │  ← 날짜
│  │░░░░░░│  ░░░░░                │  ← 가격
│  └──────┘  ░░░                  │  ← 상태
└─────────────────────────────────┘
```

```
│      │ ░░░░░░░░ │ ░░░░░░░░ │ ░░░░░░░░ │  ← 헤더 (썸네일 + 이름)
├──────┼──────────┼──────────┼──────────┤
│░░░░░ │ ░░░░░    │ ░░░░░    │ ░░░░░    │  ← 행

```

## 2. Spinner (스피너)

### 사용 컴포넌트

**Button Loading State** — 버튼 라벨을 스피너로 교체

```
[지금 예약]  →  [  ◌  ]  (loading)
```

| 속성 | 값 |
|------|-----|
| 크기 | sm=16px, md=20px, lg=20px |
| 선 굵기 | 2px |
| 색상 | 라벨과 동일 토큰 |
| 회전 속도 | 700ms / 1회전, `linear` easing |
| 전환 | Label → Spinner: `$motion.duration.normal` (250ms) |

**인라인 스피너** — 추가 로딩 표시

```
┌─────────────────────────────────┐
│  [Card]                         │
│  [Card]                         │
│  [Card]                         │
│                                 │
│          ◌ 불러오는 중...        │  ← 인라인 스피너 (24px)
│                                 │
└─────────────────────────────────┘
```

---

## 3. Pull to Refresh (당겨서 새로고침)

### 모바일 전용

```
                ◌                  ← 스피너 (당기면 표시)
┌─────────────────────────────────┐
│  [리스트 콘텐츠]                 │
│  ...                            │
└─────────────────────────────────┘
```

| 속성 | 값 |
|------|-----|
| 트리거 거리 | 64px (당김) |
| 스피너 크기 | 24px |
| 스피너 색상 | `$color.fg.brand` |

---

## 4. Infinite Scroll (무한 스크롤)

검색 결과 리스트 하단에 도달하면 자동으로 다음 페이지를 로딩합니다.

```
[Card]
[Card]
[Card]
─── ◌ 더 불러오는 중 ───          ← 하단 로딩 영역
```

| 속성 | 값 |
|------|-----|
| 트리거 | 하단에서 200px 지점 도달 시 |
| 로딩 표시 | 24px 스피너 + 텍스트 |
| 마지막 페이지 | "모든 결과를 확인했습니다" 텍스트 |

---

## 로딩 시간별 전략

| 시간 | 전략 |
|------|------|
| < 300ms | 아무것도 표시하지 않음 (깜빡임 방지) |
| 300ms ~ 1s | 스피너 또는 프로그레스 표시 |
| 1s ~ 3s | Skeleton 패턴 적용 |
| > 3s | Skeleton + 예상 시간 안내 ("잠시만 기다려 주세요") |
| > 10s | 타임아웃 안내 + 재시도 버튼 |

---

## 접근성

| 요구사항 | 구현 |
|---------|------|
| 로딩 상태 알림 | `aria-busy="true"` on container |
| 스피너 설명 | `aria-label="로딩 중"` |
| 스켈레톤 | `aria-hidden="true"` (장식적 요소) |
| 완료 알림 | `aria-live="polite"` 영역에 "로딩 완료" 알림 |
| Reduced Motion | 스켈레톤 펄스 애니메이션 정지, 스피너 정적 표시 |

---

## Do & Don't

### DO

- ✅ 콘텐츠 레이아웃을 예측 가능하게 유지합니다 (Skeleton은 실제 레이아웃과 동일)
- ✅ 300ms 미만 로딩은 표시하지 않습니다 (깜빡임 방지)
- ✅ 장시간 로딩 시 취소 수단을 제공합니다

### DON'T

- ❌ 풀 스크린 스피너로 전체 화면을 가리지 않습니다
- ❌ 로딩 중 사용자 입력을 무시하지 않습니다 (가능한 영역은 인터랙션 허용)
- ❌ 스켈레톤에 실제 데이터와 다른 레이아웃을 사용하지 않습니다 (CLS 방지)
