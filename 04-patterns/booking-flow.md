# Booking Flow (예약 플로우)

> 사용자가 렌터카 또는 숙박을 선택하고 예약을 완료하는 플로우

---

## 개요

Booking Flow는 사용자가 검색 결과에서 상품을 선택한 후 **예약 완료까지 진행하는 핵심 전환 퍼널**입니다. 단계별로 필요한 정보만 수집하는 프로그레시브 디스클로저(Progressive Disclosure) 방식을 사용합니다.

---

## 플로우 단계

```
Step 1          Step 2          Step 3          Step 4          Step 5
상품 상세   →   옵션 선택   →   정보 입력   →   결제       →   확인
(탐색)         (구체화)        (수집)         (결정)         (완료)
```

---

## Step 1: 상품 상세

### 화면 구성

```
┌─────────────────────────────────┐
│  [이미지 갤러리 / 캐러셀]        │
├─────────────────────────────────┤
│  현대 아반떼 2026               │  ← $typography.title2
│  📍 서울 강남 · ⭐ 4.8 (128건)   │  ← $typography.body2
├─────────────────────────────────┤
│  상세 정보 섹션                  │  ← section-gap: 24px
│  ├─ 차량 특징 / 숙소 편의시설    │
│  ├─ 이용 조건                    │
│  └─ 리뷰                        │
├─────────────────────────────────┤
│  ┌─────────────────────────┐   │
│  │  ₩45,000/일  [예약하기]  │   │  ← Sticky footer
│  └─────────────────────────┘   │     Button(primary, lg)
└─────────────────────────────────┘
```

### 핵심 원칙

- 가격은 항상 `$typography.price-large`로 눈에 띄게 표시
- "예약하기" 버튼은 스크롤과 무관하게 하단 고정 (sticky)
- 핵심 정보(가격, 평점, 위치)는 스크롤 없이 첫 화면에 노출

---

## Step 2: 옵션 선택

### 사용 컴포넌트

| 컴포넌트 | 용도 |
|----------|------|
| **Card** (`variant="base"`) | 옵션 카드 (보험, 부가 서비스) |
| **BottomSheet** | 옵션 상세 설명 |
| **Button** | 다음 단계 진행 |

### 옵션 유형

```
렌터카:
├─ 보험 선택 (기본/종합/완전자차)    → Card 라디오 선택
├─ 부가 장비 (네비/카시트/와이파이)   → Card 체크박스 선택
└─ 반납 위치 (동일/타지역)           → TextField 또는 선택

숙박:
├─ 객실 타입 (스탠다드/디럭스/스위트) → Card 라디오 선택
├─ 부가 서비스 (조식/레이트체크아웃)   → Card 체크박스 선택
└─ 특별 요청 (TextField)
```

---

## Step 3: 정보 입력

### 사용 컴포넌트

| 컴포넌트 | 용도 |
|----------|------|
| **TextField** | 이름, 이메일, 전화번호, 운전면허 번호 |
| **Button** | 폼 제출 |

### 폼 구성

```
┌─────────────────────────────────┐
│  예약자 정보                     │  ← $typography.title3
│                                 │
│  이름 *                         │  ← label
│  ┌─────────────────────────┐   │
│  │  홍길동                   │   │  ← TextField
│  └─────────────────────────┘   │
│  ← 16px gap →                  │  ← $dimension.spacing-y.component-gap-wide
│  이메일 *                       │
│  ┌─────────────────────────┐   │
│  │  hong@email.com          │   │
│  └─────────────────────────┘   │
│  올바른 이메일을 입력해 주세요     │  ← Helper text (or error)
│                                 │
│  연락처 *                       │
│  ┌─────────────────────────┐   │
│  │  010-1234-5678           │   │
│  └─────────────────────────┘   │
├─────────────────────────────────┤
│  ┌─────────────────────────┐   │
│  │      다음 단계            │   │  ← Button(primary, lg, fill)
│  └─────────────────────────┘   │
└─────────────────────────────────┘
```

### 유효성 검사 규칙

| 필드 | 검증 시점 | 규칙 |
|------|-----------|------|
| 이름 | blur | 2자 이상, 특수문자 금지 |
| 이메일 | blur | 이메일 형식 (RFC 5322) |
| 전화번호 | 입력 중 | 숫자 + 하이픈만, 자동 포맷팅 |
| 운전면허 | blur | 국가별 형식 검증 |

→ 상세 유효성 검사 패턴은 [Form Input](./form-input.md) 참조

## Step 4: 결제

### 화면 구성

```
┌─────────────────────────────────┐
│  결제 정보                       │  ← $typography.title3
│                                 │
│  ┌─ 예약 요약 ─────────────┐   │
│  │  현대 아반떼              │   │  ← Card(booking) 간소화
│  │  3/15 ~ 3/18 (3일)       │   │
│  │  기본 보험 포함            │   │
│  │─────────────────────────│   │
│  │  대여료      ₩135,000    │   │
│  │  보험료       ₩30,000    │   │
│  │  할인         -₩15,000   │   │  ← $color.fg.positive
│  │─────────────────────────│   │
│  │  총 결제액    ₩150,000   │   │  ← $typography.price-large
│  └─────────────────────────┘   │
│                                 │
│  결제 수단                       │
│  [신용카드] [간편결제] [계좌이체]  │
│                                 │
├─────────────────────────────────┤
│  ┌─────────────────────────┐   │
│  │   ₩150,000 결제하기      │   │  ← Button(primary, lg, fill)
│  └─────────────────────────┘   │
└─────────────────────────────────┘
```

### 결제 버튼 규칙

- 라벨에 **금액을 포함**합니다: "₩150,000 결제하기"
- 결제 처리 중 `loading` 상태 활성
- 중복 결제 방지: loading 중 버튼 인터랙션 차단

## Step 5: 예약 확인

### 성공 화면

```
┌─────────────────────────────────┐
│                                 │
│         ✅                      │  ← ic_status_check, $icon.xl
│    예약이 확정되었습니다           │  ← $typography.title2
│                                 │
│  ┌─────────────────────────┐   │
│  │  예약 번호: CR-20260315  │   │  ← Card(booking)
│  │  현대 아반떼              │   │
│  │  3/15(일) 10:00          │   │
│  │   ~ 3/18(수) 10:00       │   │
│  │  서울 강남 영업소          │   │
│  │  ₩150,000               │   │
│  │  상태: 확정 ✓             │   │  ← status badge (positive)
│  └─────────────────────────┘   │
│                                 │
│  [예약 상세 보기]   [홈으로]     │  ← Primary / Secondary
└─────────────────────────────────┘
```

### 실패 화면

```
┌─────────────────────────────────┐
│                                 │
│         ⚠️                      │  ← ic_status_warning, $icon.xl
│    결제에 실패했습니다            │  ← $typography.title2
│    카드 정보를 확인해 주세요      │  ← $typography.body1
│                                 │
│  [다시 시도]   [다른 결제 수단]   │
└─────────────────────────────────┘
```

## 진행 표시 (Progress Indicator)

### 모바일

```
Step 1      Step 2      Step 3      Step 4
  ●──────────●──────────○──────────○
 상세        옵션       정보입력     결제

● 완료: $color.bg.brand
○ 미완료: $color.stroke.neutral
─ 연결선: $color.stroke.neutral (미완료) / $color.bg.brand (완료)
```

### 데스크톱

좌측 사이드바에 단계별 진행 상태를 표시하거나, 상단 Breadcrumb 형태로 표시합니다.

---

## 이탈 방지

| 상황 | 대응 |
|------|------|
| 폼 입력 중 뒤로가기 | Dialog(confirm): "입력한 정보가 사라집니다. 나가시겠습니까?" |
| 결제 진행 중 이탈 | Dialog(alert): "결제가 진행 중입니다. 잠시만 기다려 주세요." |
| 세션 만료 | Dialog(alert): "세션이 만료되었습니다. 다시 로그인해 주세요." |
| 재고 소진 | Dialog(alert): "선택한 차량이 마감되었습니다." → 검색 결과로 복귀 |
